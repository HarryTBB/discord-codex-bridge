# Discord Codex Bridge v0.1.0

## Summary

`v0.1.0` is the first public release of **Discord Codex Bridge**, a Discord slash-command service for securely operating Codex CLI from a private Discord server.

This release focuses on:

- project-scoped access control
- mobile-friendly Discord output
- resumable Codex sessions
- admin-managed user onboarding and permissions
- practical deployment on Linux or WSL

## Highlights

### Project-scoped execution

Users can be assigned one or more top-level projects, and all file operations are restricted to the currently active project.

### Resumable Codex sessions

The bridge can bind a Discord scope to the latest Codex session in the current project using `/resume`.

### Mobile-first Discord experience

Commands such as `/status`, `/ls`, and `/ask` render cleanly in Discord, with structured code panels and progressive status updates for longer-running tasks.

### Admin controls

Admins can:

- add and remove users
- grant project access
- set permission level
- manage users by mention or by explicit user ID

### User-specific runtime settings

Users can manage their own:

- model override
- reasoning effort

The bridge also exposes the latest available Codex rate-limit estimate via `/rate-limits`.

## Deployment Notes

This release is intended for:

- private Discord servers
- Linux or WSL deployments
- operators who already have Codex CLI installed and authenticated

The repository includes:

- `.env.example` for environment setup
- `deploy/discord-codex-bridge.service` for `systemd`
- a fully documented `README.md`

## Security Notes

- `.env`, runtime state, and dependency directories are intentionally excluded from version control
- per-project ACL is the preferred access model
- `full-access` should be reserved for trusted operators only

## Known Operational Notes

- auto-onboarding on guild join requires Discord **Server Members Intent**
- slash commands must be registered per guild using `npm run register`
- `/resume` depends on a valid `CODEX_HOME` and prior Codex sessions
